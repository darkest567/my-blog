Spring概念

Spring 是一个开源的 Java 框架，它为开发者提供了全面的基础设施支持，简化了 Java 企业级应用程序的开发过程。Spring 框架的核心理念是“控制反转”（IoC）和“面向切面编程”（AOP），并且它提供了许多功能和模块，帮助开发人员构建高效、可扩展、可维护的应用。

  1. Spring引入带来了哪些好处？

    • 简化开发，降低企业级开发的复杂性
    • 框架整合，高效整合其他技术，提高企业级应用开发和运行效率。

  2. 什么是AOP？

    面向切面编程 是一种编程范式，它允许将程序的不同部分（横切关注点）进行分离，使代码更加模块化。横切关注点是指那些贯穿整个应用程序的功能，比如日志记录、事务管理、安全控制等。

  3. 什么是IOC？

    控制反转（IOC） 是指 控制权 的转移，而在 IoC 中，控制权被交给容器，以达到降低程序耦合度的作用。

  4. 什么是Bean？

Bean 就是 Spring 容器中一个受控的对象，Spring 容器负责其生命周期、依赖关系的管理以及其它的一些功能。

  1. Bean是如何创建的？实例化bean的三种方式是什么？

    Bean通过调用方法在Spring容器创建。

    实例化bean的方法：构造方法（空参），静态工厂，实例工厂

  2. 普通对象怎么才能变成bean？

    将普通对象变为bean需要将对象的创建和控制改为在spring中进行。

  3. 描述一下Bean生命周期

    在bean被实例化时bean的生命周期开始，当spring容器关闭时bean被销毁。

    可以用close和registerShtdownHook方法关闭Spring以结束Bean的生命周期。

依赖注入

DI，在容器中建立bean和bean之间的依赖关系的整个过程，称为依赖注入。

  1. setter注入

    通过set方法将内容注入到bean中。

    (ref用作引用,value用作具体值)

  2. 构造器注入

    通过有参构造方法注入

    可以通过类型或顺序注入不再使用参数名，用来降低耦合度

  3. 自动注入

    自动注入依然依赖set方法

    • 自动装配用于引用类型依赖注入，不能对简单类型进行操作。
    • 使用按类型装配时(byType)必须保障容器中相同类型的bean唯一，推荐使用。
    • 使用按名称装配时(byName)必须保障容器中具有指定名称的bean，因变量名与配置耦合，不推荐使用。
    • 自动装配优先级低于setter注入与构造器注入，同时出现时自动装配配置失效。
